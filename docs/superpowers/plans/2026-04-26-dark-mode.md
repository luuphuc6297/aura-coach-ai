# Dark Mode Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship a real, end-to-end dark mode that flips every screen — not just Material defaults — driven by the existing `SettingsProvider.themeMode` toggle.

**Architecture:** Define `ClayPalette extends ThemeExtension<ClayPalette>` with light + dark variants for the 6 surface tokens (background, surface, surfaceAlt, border, text, textMuted, shadow). Accent tokens (teal, coral, gold, purple, success, error) stay constant across modes. Migrate ~838 hardcoded `AppColors.<surface>` references to `context.clay.<surface>` lookups so widgets re-resolve when `MaterialApp.themeMode` flips. Convert `AppShadows` constants to context-aware getters because shadow color changes per mode.

**Tech Stack:** Flutter 3.x, Provider, ThemeExtension API, existing SettingsProvider/SharedPreferences wiring.

**Scope (measured):**
- 67 `AppColors.cream` refs → `context.clay.background`
- 103 `AppColors.clayWhite` → `context.clay.surface`
- 34 `AppColors.clayBeige` → `context.clay.surfaceAlt`
- 124 `AppColors.clayBorder` → `context.clay.border`
- 4 `AppColors.clayShadow` → `context.clay.shadow`
- 248 `AppColors.warmDark` → `context.clay.text`
- 188 `AppColors.warmMuted` → `context.clay.textMuted`
- 70 `AppColors.warmLight` → `context.clay.textFaint`
- 60 `AppShadows.<X>` refs → `AppShadows.X(context)` getters
- ~30 const constructors will need to drop `const` (mechanical)
- ~92 files touched
- Plus restore the dark-mode picker in `settings_screen.dart`

```dart
// lib/core/theme/clay_palette.dart
import 'package:flutter/material.dart';
import 'app_colors.dart';

/// Theme-aware surface tokens. Accent colors (teal, coral, gold, purple,
/// success, error) live on AppColors and stay constant across modes —
/// only surfaces, text, borders, and shadow flip.
@immutable
class ClayPalette extends ThemeExtension<ClayPalette> {
  final Color background;
  final Color surface;
  final Color surfaceAlt;
  final Color border;
  final Color text;
  final Color textMuted;
  final Color textFaint;
  final Color shadow;

  const ClayPalette({
    required this.background,
    required this.surface,
    required this.surfaceAlt,
    required this.border,
    required this.text,
    required this.textMuted,
    required this.textFaint,
    required this.shadow,
  });

  static const light = ClayPalette(
    background: AppColors.cream,
    surface: AppColors.clayWhite,
    surfaceAlt: AppColors.clayBeige,
    border: AppColors.clayBorder,
    text: AppColors.warmDark,
    textMuted: AppColors.warmMuted,
    textFaint: AppColors.warmLight,
    shadow: AppColors.clayShadow,
  );

  // Dark variant stays inside the WARM clay hue family — the same purple-blue
  // axis as light mode, just inverted in luminosity. surface is literally
  // AppColors.warmDark (the light-mode text color), text is AppColors.cream
  // (the light-mode background). Keeps clay aesthetic consistent across modes.
  static const dark = ClayPalette(
    background: Color(0xFF1A1C2E), // deep warm navy, slightly darker than warmDark
    surface: Color(0xFF2D3047),    // = AppColors.warmDark
    surfaceAlt: Color(0xFF3A3D55), // lifted warm fill (input bg)
    border: Color(0xFF4A4D62),     // mid warm purple-gray
    text: Color(0xFFFFF8F0),       // = AppColors.cream
    textMuted: Color(0xFFB8B6CC),  // warm muted-cream
    textFaint: Color(0xFF82849A),  // warm faint
    shadow: Color(0x73000000),     // 45% black — bold drop on warm dark surface
  );

  @override
  ClayPalette copyWith({
    Color? background,
    Color? surface,
    Color? surfaceAlt,
    Color? border,
    Color? text,
    Color? textMuted,
    Color? textFaint,
    Color? shadow,
  }) {
    return ClayPalette(
      background: background ?? this.background,
      surface: surface ?? this.surface,
      surfaceAlt: surfaceAlt ?? this.surfaceAlt,
      border: border ?? this.border,
      text: text ?? this.text,
      textMuted: textMuted ?? this.textMuted,
      textFaint: textFaint ?? this.textFaint,
      shadow: shadow ?? this.shadow,
    );
  }

  @override
  ClayPalette lerp(ThemeExtension<ClayPalette>? other, double t) {
    if (other is! ClayPalette) return this;
    return ClayPalette(
      background: Color.lerp(background, other.background, t)!,
      surface: Color.lerp(surface, other.surface, t)!,
      surfaceAlt: Color.lerp(surfaceAlt, other.surfaceAlt, t)!,
      border: Color.lerp(border, other.border, t)!,
      text: Color.lerp(text, other.text, t)!,
      textMuted: Color.lerp(textMuted, other.textMuted, t)!,
      textFaint: Color.lerp(textFaint, other.textFaint, t)!,
      shadow: Color.lerp(shadow, other.shadow, t)!,
    );
  }
}

/// Ergonomic accessor: `context.clay.surface` instead of
/// `Theme.of(context).extension<ClayPalette>()!.surface`.
extension ClayPaletteContext on BuildContext {
  ClayPalette get clay => Theme.of(this).extension<ClayPalette>()!;
}
```

## Task 2: Convert AppShadows to context-aware getters

**Files:**
- Modify: `lib/core/theme/app_shadows.dart`

The shadow color must flip per mode, but `AppShadows.clay` is currently `static const` so it can't read context. Convert to a class with static getter methods that take `BuildContext`.

- [ ] **Step 1: Refactor AppShadows**

Read current `lib/core/theme/app_shadows.dart` to find every constant. Replace each `static const X = [...]` with a static method `static List<BoxShadow> X(BuildContext context)`. Use `context.clay.shadow` for the color. Example:

```dart
import 'package:flutter/material.dart';
import 'clay_palette.dart';

abstract final class AppShadows {
  static List<BoxShadow> clay(BuildContext context) => [
        BoxShadow(color: context.clay.shadow, offset: const Offset(3, 3)),
      ];

  static List<BoxShadow> clayPressed(BuildContext context) => [
        BoxShadow(color: context.clay.shadow, offset: const Offset(1, 1)),
      ];

  // clayBold uses warmDark in light mode for emphasis — keep that intent
  // but drive it from the palette's `text` token so it inverts in dark.
  static List<BoxShadow> clayBold(BuildContext context) => [
        BoxShadow(color: context.clay.text, offset: const Offset(3, 3)),
      ];

  static List<BoxShadow> clayBoldPressed(BuildContext context) => [
        BoxShadow(color: context.clay.text, offset: const Offset(1, 1)),
      ];

  static List<BoxShadow> card(BuildContext context) => [
        BoxShadow(
          color: context.clay.shadow.withValues(alpha: 0.04),
          offset: const Offset(0, 2),
          blurRadius: 8,
        ),
      ];

  // ... convert remaining: soft, lifted, accentClay, etc.
}
```

```dart
String _themeLabel(ThemeMode mode) {
  switch (mode) {
    case ThemeMode.system: return 'System';
    case ThemeMode.light:  return 'Light';
    case ThemeMode.dark:   return 'Dark';
  }
}

Future<void> _pickThemeMode(BuildContext context) async {
  final settings = context.read<SettingsProvider>();
  final picked = await showModalBottomSheet<ThemeMode>(
    context: context,
    backgroundColor: context.clay.surface,
    shape: RoundedRectangleBorder(
      borderRadius: BorderRadius.vertical(top: Radius.circular(20)),
    ),
    builder: (ctx) => SafeArea(
      child: Column(
        mainAxisSize: MainAxisSize.min,
        children: [
          for (final mode in ThemeMode.values)
            ListTile(
              leading: Icon(_themeIcon(mode), color: AppColors.purple),
              title: Text(_themeLabel(mode)),
              trailing: settings.themeMode == mode
                  ? const Icon(Icons.check, color: AppColors.purple)
                  : null,
              onTap: () => Navigator.of(ctx).pop(mode),
            ),
        ],
      ),
    ),
  );
  if (picked == null) return;
  await settings.setThemeMode(picked);
}

IconData _themeIcon(ThemeMode mode) {
  switch (mode) {
    case ThemeMode.system: return Icons.brightness_auto_rounded;
    case ThemeMode.light:  return Icons.light_mode_rounded;
    case ThemeMode.dark:   return Icons.dark_mode_rounded;
  }
}
```

## Risks + Mitigations

| Risk | Mitigation |
|------|------------|
| `const` stripping introduces accidental rebuilds | Don't worry about perf in this pass — Flutter rebuilds are cheap; profile only if visible jank surfaces |
| CustomPainters miss palette updates | All `shouldRepaint` overrides must compare palette equality |
| Cloudinary asset images look out of place in dark | Wrap illustration containers in `surfaceAlt` background tint, or accept as-is — note in PR |
| Some color tokens used for non-surface meaning (e.g. `warmDark` as a "dark accent") will lose intent in dark mode | Audit during Task 7; if a ref is semantically not a surface color, keep it pointing at the static `AppColors.warmDark` and add a comment |
| Existing `_ComingSoonRow` widget — leave or remove? | Leave the class; just stop using it for theme. Vietnamese task may still need it as a transitional state |
